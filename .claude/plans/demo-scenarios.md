# 演示场景升级方案

## 目标

把 `proposal-demo` 页面从"技术验证"升级为"能打动客户的演示工具"。

**核心体验**：设计师打开演示页面 → 客户看到一个真实的客厅 → 点击"影院模式" → 灯光渐暗，窗帘缓缓拉上 → "这就是你回家后的样子"。

---

## 现状分析

**已经能跑的**：
- ProposalLayout 全套 UI（场景栏、子系统栏、设备信息卡、报价面板）
- DeviceRenderer + DeviceLight（真实 PointLight，带 lerp 平滑过渡）
- LightCone 粒子光锥（灯光关闭时渐隐）
- CurtainPanel 动画（开合 + 帘布飘动）
- 4 个场景（回家/影院/离家/晨间），applyScene 一键批量更新
- 11 个设备（4 筒灯 + 2 面板 + 1 传感器 + 1 窗帘 + 1 温控 + 1 AP + 1 门锁）

**关键不足**：
1. **房间是 box 堆出来的** — 沙发、茶几、电视柜全是方块，毫无质感
2. **只有 4 盏筒灯** — 灯光类型单一，缺少灯带和壁灯的氛围层次
3. **场景不够真实** — 缺少晚餐、阅读、会客等日常场景
4. **房间只有一个** — 没有卧室，不能展示多空间联动
5. **初始状态没有"入场感"** — 页面加载后直接全亮，缺少 wow moment

---

## 实施计划

### Phase 1: 客厅家具换真实 GLB 模型

**改动文件**: `apps/editor/app/proposal-demo/page.tsx`

用 `useGLTF` 加载 catalog 中已有的 GLB 模型，替换手写 box 几何体：

| 家具 | 替换方案 | 模型 ID | 尺寸 |
|------|---------|---------|------|
| 沙发 | GLB 模型 | `sofa` | 2.5×0.8×1.5m |
| 茶几 | GLB 模型 | `coffee-table` | 2×0.4×1.5m |
| 电视柜 | GLB 模型 | `tv-stand` | 2×0.4×0.5m |
| 电视 | GLB 模型 | `flat-screen-tv` | 1.4×0.8×0.06m |
| 台灯 | GLB 模型 | `table-lamp` | 0.5×0.8×1m |
| 书架 | GLB 模型 | `bookshelf` | 1×2×0.5m |

保留手写几何体的部分：
- 墙壁、地板、天花板（结构体无需 GLB）
- 踢脚线、装饰面板（简单几何即可）

实现方式：在 demo 页面内新增 `FurnitureModel` 组件，用 drei 的 `useGLTF` 直接加载 `/items/{id}/model.glb`。

### Phase 2: 扩展设备种类 — 增加灯光层次

在现有 4 盏筒灯基础上，增加：

| 设备 | 位置 | 目的 |
|------|------|------|
| 灯带 ×2 | 电视柜上方/下方 | 氛围光，影院模式核心 |
| 壁灯 ×1 | 沙发区右侧墙 | 阅读辅助光 |

这样灯光分 3 层：
- **主照明**（筒灯 ×4）— 整体照亮
- **氛围光**（灯带 ×2）— 柔和底光
- **功能光**（壁灯 ×1）— 局部阅读

### Phase 3: 6 个真实场景

| 场景 | 灯光 | 窗帘 | 情感 |
|------|------|------|------|
| 🏠 回家模式 | 筒灯 80% 暖白 3000K + 灯带 50% | 开 | 温暖、放松 |
| 🎬 影院模式 | 筒灯全灭 + 灯带 15% 暖色 2700K | 关 | 沉浸、幽暗 |
| 🍽️ 晚餐模式 | 主灯 A/B 60% 3200K + 灯带 40% 2700K + 其他灭 | 关 | 浪漫、私密 |
| 📖 阅读模式 | 壁灯 100% 4500K + 沙发区灯 40% 4000K + 其他灭 | 开 | 安静、护眼 |
| 👋 会客模式 | 全部 100% 自然白 5000K | 开 | 明亮、大方 |
| 🚪 离家模式 | 全灭 | 关 | 安全、节能 |

### Phase 4: 入场动画

页面加载时初始状态为"离家模式"（全暗），然后自动执行"回家模式"：
1. 0ms: 页面加载，全黑
2. 500ms: 自动 applyScene("回家模式")
3. 灯光 lerp 渐入（已有 DeviceLight 动画支持，约 500ms）
4. 窗帘同步拉开（CurtainPanel 已有过渡动画）

客户看到的效果：**黑屏 → 灯光渐亮 → 窗帘拉开 → "欢迎回家"**

实现：在 useEffect 末尾 setTimeout 500ms 后调用 applyScene(homeSceneId)，初始设备状态设为全灭。

### Phase 5: 卧室场景（可选扩展）

如果客厅演示效果好，再加卧室：

**结构**：在同一个 demo 页面内，增加第二个 Level（F2 主卧），通过 ProposalLayout 的面包屑切换。

**家具**：double-bed (2×0.8×2.5m), bedside-table ×2, closet, table-lamp ×2

**设备**：筒灯 ×2, 灯带 ×1 (床头), 窗帘 ×1, 温控 ×1

**场景**：起床模式 / 睡前模式 / 夜灯模式

## 不做什么

- ❌ 不改 DeviceRenderer / DeviceLight 等渲染管线 — 已经够好
- ❌ 不做多房间联动 — 当前架构每个 Level 独立
- ❌ 不做声音效果 — 超出范围
- ❌ 不改 ProposalLayout UI — 已经完善
