# VilHil Studio 页面流程与导航架构

> VilHil Studio 不是一个编辑器，是一个**工作台**。
> 设计师按阶段完成工作，每个阶段有独立的工具集和视图。

---

## 一、工作阶段模型

设计师的工作流程是线性的，但可以自由回退：

```
建筑 → 布置 → 智能 → 场景 → 演示 → 交付
  ↑_______________________________↓（可以随时回退修改）
```

### 6 个工作阶段

| 阶段 | 名称 | 用户在做什么 | 核心工具 |
|------|------|------------|---------|
| 1 | **建筑** | 画户型：墙/门窗/楼板/楼层 | Pascal Structure 工具 |
| 2 | **布置** | 放家具：沙发/床/桌椅/电器 | Pascal Furnish 工具 |
| 3 | **智能** | 放设备：灯/面板/传感器/AP + 配参数 | VilHil 设备目录 + 属性面板 |
| 4 | **场景** | 编排场景：回家模式/离家模式/影院模式 | 场景编辑器（时间线） |
| 5 | **演示** | 给客户看：全屏 3D + 设备操控 + 场景播放 | Proposal 展示 UI |
| 6 | **交付** | 出文件：设备清单/报价单/方案链接 | 导出面板 |

## 二、导航架构

### 顶部主导航（阶段切换）

```
┌──────────────────────────────────────────────────────────────────┐
│  VilHil   [建筑] [布置] [智能] [场景]  │  [演示]  │  [交付]    │
│  Studio    ← 编辑阶段（左侧） →        │ 展示模式 │ 导出面板   │
└──────────────────────────────────────────────────────────────────┘
```

**设计理由**：
- 前 4 个阶段是编辑工作（左侧聚合）
- "演示"是独立模式（全屏切换，隐藏编辑 UI）
- "交付"是弹出面板（不离开当前视图）

### 与 Pascal 现有导航的关系

Pascal 现有：`[3D] [2D] [Split]` + 左侧 `[Structure] [Furnish] [Zones]`

VilHil 改造：
- 顶部 `[3D] [2D] [Split]` 保留 — 每个阶段都可以切 2D/3D
- 左侧 Tab 变成**阶段对应的工具集**：

| 阶段 | 左侧 Tab | 内容 |
|------|---------|------|
| 建筑 | Structure | 墙/门窗/楼板/天花/楼梯/屋顶（Pascal 原有）|
| 布置 | Furnish | 家具/电器/厨卫/户外（Pascal 原有）|
| 智能 | Smart | 9 子系统设备目录 |
| 场景 | Scenes | 场景列表 + 时间线编辑器 |

### 阶段切换时发生什么

| 切换到 | 左侧栏变化 | 3D 视图变化 | 右侧面板变化 |
|--------|----------|-----------|------------|
| 建筑 | 显示 Structure 工具 | 标准建筑视图 | 选中元素的建筑属性 |
| 布置 | 显示 Furnish 目录 | 标准建筑视图 + 家具 | 选中家具的属性 |
| 智能 | 显示 9 子系统设备目录 | 建筑 + 家具 + 设备标注 | 选中设备的参数 |
| 场景 | 显示场景列表 | 建筑 + 家具 + 设备 + 动画效果 | 场景时间线编辑器 |
| 演示 | 隐藏（全屏 3D） | 全部可视化效果 + 操控交互 | 无（底部场景卡片） |
| 交付 | 显示导出选项 | 不变 | 设备清单/报价 |

## 三、每个阶段的详细 UI 布局

### 阶段 1-2：建筑 + 布置

直接使用 Pascal 原有 UI，不改动。

```
┌─────────┬──────────────────────────────────────┐
│ Structure│                                      │
│ 或       │          3D / 2D / Split             │
│ Furnish  │                                      │
│ 工具列表  │                                      │
│          │                                      │
│          ├──────────────────────────────────────┤
│          │       选中元素属性面板（右侧弹出）      │
└─────────┴──────────────────────────────────────┘
```

### 阶段 3：智能（核心新增）

```
┌─────────┬──────────────────────────────────────┬────────────┐
│ 子系统    │                                      │ 设备属性    │
│ ● 灯光   │          3D 视图                      │            │
│ ● 面板   │    （建筑 + 家具 + 设备标注）           │ 亮度 ━━━━  │
│ ● 传感器  │                                      │ 色温 ━━━━  │
│ ● 窗帘   │    选中子系统时                        │ 角度 ━━━━  │
│ ● 暖通   │    该子系统设备高亮                     │ 范围 ━━━━  │
│ ● ...    │    其他子系统设备变灰                   │            │
│          │                                      │ [删除]     │
│ ─────── │                                      │            │
│ 设备目录  │                                      │            │
│ 筒灯     │                                      │            │
│ 射灯     │                                      │            │
│ 灯带     │          底部：子系统图例栏             │            │
└─────────┴──────────────────────────────────────┴────────────┘
```

**交互流程**：
1. 左侧选子系统 → 3D 中该子系统设备高亮
2. 左侧展开设备目录 → 拖拽设备到 3D 场景
3. 设备吸附到天花/墙面/地面
4. 右侧自动弹出属性面板

### 阶段 4：场景（核心新增）

```
┌─────────┬──────────────────────────────────────┬────────────┐
│ 场景列表  │                                      │ 时间线编辑  │
│          │          3D 视图                      │            │
│ + 新场景  │    （全部设备 + 动画效果）              │ 0s ──────  │
│          │                                      │ 筒灯 ▬▬▬▬ │
│ 回家模式  │                                      │ 灯带  ▬▬▬ │
│ 离家模式  │    播放场景时                          │ 窗帘   ▬▬ │
│ 影院模式  │    设备按时间线依次变化                  │ 空调    ▬ │
│ 睡眠模式  │                                      │            │
│          │                                      │ [播放预览] │
│          │                                      │ [保存场景] │
│          │                                      │            │
│          │          底部：播放控制栏               │            │
│          │          [◀] [▶] [⏸] 0:00 / 0:05    │            │
└─────────┴──────────────────────────────────────┴────────────┘
```

**交互流程**：
1. 左侧创建/选择场景
2. 右侧时间线：拖拽设备状态变化到时间轴
3. 点击播放预览 → 3D 中设备按序联动
4. 满意后保存场景

### 阶段 5：演示（全屏模式）

```
┌──────────────────────────────────────────────────────────────┐
│ [← 返回编辑]          项目名称             [子系统筛选] [设置]│
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ ● 灯光                                                      │
│ ● 面板          全屏 3D 场景                                  │
│ ● 传感器        所有设备活着（L1）                             │
│ ● 窗帘          可点击操控（L2）                               │
│ ● 暖通          场景联动（L3）                                │
│ ● ...                                                       │
│                                                              │
│                                                              │
│              [回家模式] [离家模式] [影院模式]                   │
└──────────────────────────────────────────────────────────────┘
```

**与编辑模式的区别**：
- 没有左侧工具栏 → 换成子系统图标栏
- 没有属性面板
- 底部是场景卡片
- 设备可点击操控但不可移动/删除
- 全屏沉浸感

### 阶段 6：交付（弹出面板）

不离开当前视图，右侧弹出导出面板：

```
┌──────────────────────────┬──────────────────┐
│                          │ 方案交付          │
│      当前 3D 视图         │                  │
│      （不变）             │ 设备清单          │
│                          │ ├ 灯光 × 8       │
│                          │ ├ 面板 × 4       │
│                          │ ├ 传感器 × 3     │
│                          │ └ AP × 2         │
│                          │                  │
│                          │ [导出 Excel]     │
│                          │ [导出 PDF]       │
│                          │ [生成分享链接]    │
└──────────────────────────┴──────────────────┘
```

## 四、技术实现方案

### URL 路由

```
/                          → 项目列表（未来）
/editor/:projectId         → 编辑模式（建筑/布置/智能/场景）
/proposal/:projectId       → 展示模式
/share/:shareToken         → 客户分享链接（只读展示）
```

### 阶段状态

```typescript
// packages/editor/src/store/use-editor.ts
interface EditorState {
  phase: 'structure' | 'furnish' | 'smart' | 'scene'  // 当前阶段
  // ... existing fields
}
```

### 阶段切换组件

```
apps/editor/
  app/
    page.tsx                → 编辑模式主页
    proposal/page.tsx       → 展示模式
    share/[token]/page.tsx  → 分享链接
packages/editor/
  components/
    editor/
      phase-nav.tsx         → 顶部阶段导航栏（新增）
    proposal/               → 展示模式 UI（已有骨架）
    smart/                  → 智能阶段 UI（新增）
      subsystem-panel.tsx   → 子系统选择 + 设备目录
      device-properties.tsx → 设备属性面板
    scene/                  → 场景阶段 UI（新增）
      scene-list.tsx        → 场景列表
      timeline-editor.tsx   → 时间线编辑器
      playback-controls.tsx → 播放控制栏
    delivery/               → 交付面板（新增）
      export-panel.tsx      → 导出选项
      device-summary.tsx    → 设备清单
```

## 五、阶段间的数据流

```
建筑阶段 → 产出：rooms[], walls[], openings[]
    ↓
布置阶段 → 产出：furniture[]（位置、尺寸）
    ↓
智能阶段 → 产出：devices[]（位置、参数、子系统）
    ↓    → 消费：rooms（设备关联到房间）、walls（面板吸附墙面）、openings（窗帘关联窗户）
    ↓
场景阶段 → 产出：scenes[]（时间线、设备状态快照）
    ↓    → 消费：devices（编排哪些设备参与场景）
    ↓
演示阶段 → 消费：全部数据（建筑 + 家具 + 设备 + 场景）
    ↓    → 产出：无（只读展示）
    ↓
交付阶段 → 消费：devices（生成清单）、scenes（方案说明）
         → 产出：Excel / PDF / 分享链接
```

## 六、开发优先级

| 优先级 | 阶段 | 状态 | Sprint |
|--------|------|------|--------|
| P0 | 建筑 | ✅ Pascal 已完成 | — |
| P0 | 布置 | ✅ Pascal 已完成 | — |
| P0 | 智能 | 骨架已有，需完善 | S1 |
| P1 | 演示 | 骨架已有，需完善 | S2 |
| P1 | 场景 | 未开始 | S3 |
| P2 | 交付 | 未开始 | S4 |
| — | 阶段导航 | 未开始 | S1（简单版）|

### S1 简单版导航

不做完整的 6 阶段导航。在 Furnish 的分类列表里加一个 **"智能"** 子分类（跟家具/电器/厨卫并列），不另起独立 Tab。

展示模式通过顶栏的 Preview 按钮进入（Pascal 已有）。

**不要一开始就做一个巨大的导航框架。先让一个功能可用，再扩展。6 阶段模型是远期目标，不是 S1 的任务。**

### 关于 WebGPU 与微信

S5 的"客户在微信里打开方案链接"场景可能遇到 WebGPU 不支持的问题。分享链接的 Viewer 可能需要单独做一个 WebGL 版本的轻量渲染器。这是 S5 的问题，当前不处理。
