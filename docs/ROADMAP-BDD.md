# VilHil Studio 开发路线图（BDD 驱动）

> 每个 Sprint 以用户场景（Given/When/Then）定义目标，不以技术任务定义。
> 完成标准：用户能走通整个场景，不是"代码写完了"。

---

## Sprint 1：设计师能画一个房间并放一盏灯（1 周）

### 用户场景

```
场景：设计师第一次使用 VilHil Studio
Given 设计师打开 studio.vilhil.cn
When  他画了 4 面墙围成一个客厅
      他从设备目录拖了一盏筒灯到天花板
      他切换到展示模式
Then  他看到一个白色沙盘房间
      天花板上有一盏灯在发光，光锥向下照射
      他觉得"这就是我想给客户展示的效果"
```

### 技术任务

| # | 任务 | 包 | 验证标准 |
|---|------|-----|---------|
| 1 | 设备目录 UI 接入编辑器工具栏 | editor | 左侧栏有"设备"Tab，点开看到 9 子系统设备列表 |
| 2 | 设备拖拽放置到 3D 场景 | editor/tools | 从目录拖灯具到场景，自动吸附天花板 |
| 3 | 设备渲染器基础 | viewer/renderers | 放置的灯具在 3D 中可见（简单几何体 + 子系统色） |
| 4 | 灯具光效（L1 常驻可视化） | viewer/effects | 灯具发出可见光锥 + PointLight 照亮地面 |
| 5 | 展示模式入口 | editor/proposal | 顶栏有"展示"按钮，点击隐藏编辑 UI，显示简洁展示界面 |
| 6 | 设备属性面板 | editor/panels | 选中灯具右侧显示：亮度/色温/光束角 |

### 不做

- 场景联动
- 设备操控（L2）
- 报价清单
- 多个 AP 覆盖

## Sprint 2：客户能在展示模式里开关灯（1 周）

### 用户场景

```
场景：设计师带客户看方案
Given 客户打开展示链接，看到 3D 客厅
When  客户点击天花板上的灯
Then  灯亮了，光锥出现，地面被照亮
      客户再点一下，灯灭了，光锥消失
      客户说"这个交互很直接"
```

```
场景：设计师调光
Given 设计师在编辑模式选中一盏灯
When  他拖动亮度滑块到 50%
Then  3D 中灯光变暗，光锥透明度降低
      他拖动色温滑块到 2700K
Then  灯光变成暖黄色
```

### 技术任务

| # | 任务 | 验证标准 |
|---|------|---------|
| 1 | 设备点击交互（L2） | 点击 3D 中的灯 → 灯开/关 |
| 2 | 亮度/色温实时联动 | 属性面板拖滑块 → 3D 光效同步变化 |
| 3 | 面板设备放置 | 在墙上放一个 4 键面板 |
| 4 | 面板按键联动 | 点面板按键 → 对应灯亮/灭 |
| 5 | 展示模式子系统栏 | 左侧 9 个子系统图标，点击筛选显示 |

## Sprint 3：一个完整的客厅方案（1-2 周）

### 用户场景

```
场景：完整的客厅智能家居方案
Given 设计师画好了一个客厅（墙+门+窗）
When  他放置了以下设备：
      - 3 盏筒灯（天花）
      - 1 条灯带（吊顶暗槽）
      - 1 个 4 键面板（墙面）
      - 1 个窗帘电机（窗户上方）
      - 1 个 PIR 传感器（天花）
      - 1 个温控面板（墙面）
      他创建了"回家模式"场景
      他切换到展示模式给客户看
Then  客户看到：
      - 筒灯在发光，光锥可见
      - 灯带在暗槽里亮着
      - 窗帘处于半开状态
      - 传感器有绿色检测锥
      - 面板上 4 个按键标注清楚
      客户点击面板"全开"→ 全部灯亮
      客户点击面板"影院"→ 灯暗 + 窗帘关
      客户说"这就是我想要的家"
```

### 技术任务

| # | 任务 | 验证标准 |
|---|------|---------|
| 1 | 窗帘设备 + 开合动画 | 窗帘电机放在窗户上方，帘布可开合 |
| 2 | 传感器 + 检测锥效果 | PIR 有 110° 检测锥 |
| 3 | 暖通 + 粒子气流 | 天花出风口有气流粒子 |
| 4 | 场景编辑器 | 创建场景 → 选设备 → 设目标状态 |
| 5 | 场景播放（L3） | 点击场景卡片 → 设备按时间线联动变化 |
| 6 | 展示模式场景栏 | 底部场景卡片，点击触发 |

## Sprint 4：多房间 + 网络覆盖（1-2 周）

### 用户场景

```
场景：160㎡ 别墅全屋方案
Given 设计师画好了 F1 户型（客厅+餐厅+厨房+卧室+卫生间）
When  他在全屋放置了 2 个 WiFi AP
      他切换到"网络"子系统视图
Then  他看到每个 AP 周围有信号覆盖效果
      覆盖区域在房间内自然显示，被墙衰减
      他能判断"这两个 AP 够不够覆盖全屋"
```

### 技术任务

| # | 任务 | 验证标准 |
|---|------|---------|
| 1 | 多 AP 信号覆盖可视化 | 多个 AP 的覆盖效果叠加 |
| 2 | 安防 — 摄像头 FOV | 摄像头有锥形视野，方向可调 |
| 3 | 安防 — 门锁状态 | 门锁红绿灯指示 |
| 4 | 架构 — KNX 管线 | 设备之间有连线表示总线关系 |
| 5 | 多楼层设备 | F1/F2 楼层切换，设备跟随 |
| 6 | 方案导出 | 导出设备清单 Excel |

## Sprint 5：商业化打磨（持续）

### 用户场景

```
场景：客户自己收到链接
Given 客户在微信里打开了设计师发的方案链接
When  页面加载完成
Then  他看到自己家的 3D 模型
      所有设备都在安静地"工作"
      底部有场景卡片可以点
      他不需要任何指导就能操作
      他截图发到家庭群说"你们看我家以后是这样的"
```

### 技术任务

- 分享链接系统（只读展示模式 + 短链）
- 移动端适配
- 加载性能优化（渐进加载/LOD）
- 品牌定制（Logo/主题色/水印）
- 方案模板（别墅/公寓/办公/商业）

---

## 开发节奏

| Sprint | 交付物 | 用户能做什么 |
|--------|--------|------------|
| S1 | 能画房间 + 放灯 | 最小可用 |
| S2 | 能开关灯 + 面板联动 | 核心体验 |
| S3 | 完整客厅方案 | 可演示 |
| S4 | 多房间 + 网络覆盖 | 可交付 |
| S5 | 分享链接 + 移动端 | 可商业化 |

**每个 Sprint 结束时，必须能给一个真实客户演示。不能演示的功能不算完成。**
